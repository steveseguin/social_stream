Understanding Variables and State in Your Event Flows

ü§î What Are State Nodes?

State nodes are like variables or memory in your event flows. They remember things between messages and can control how your flow behaves.

Think of state nodes as switches, counters, and timers that you can control with actions!

üì¶ Available State Nodes

üö¶ ON/OFF Switch (Gate)

What it does: Acts like a light switch - when ON, messages pass through. When OFF, messages stop.

Use cases:

  • Cooldowns (turn OFF after command, wait, turn ON)
  • Enable/disable features temporarily
  • Prevent spam by blocking messages
Works with these actions:
  • üö¶ Set Gate State - Flip the switch ON or OFF
  • üîÑ Reset State Node - Reset to default state
Example: 10-second cooldown
!hello ‚Üí [Switch ON?] ‚Üí Send Reply ‚Üí Turn Switch OFF ‚Üí Wait 10s ‚Üí Turn Switch ON

üî¢ Counter

What it does: Counts messages and triggers actions when reaching a target number.

Use cases:

  • Every Nth viewer gets special treatment
  • Track points or scores
  • Count attempts before triggering
Works with these actions:
  • üî¢ Set Counter Value - Set to any number (like resetting to 0)
  • ‚ûï Increment Counter - Add or subtract any amount
  • üîÑ Reset State Node - Reset to initial value
Example: Every 5th viewer celebration
!join ‚Üí [Counter +1] ‚Üí [At 5?] ‚Üí Special Welcome ‚Üí Auto-reset to 0

  • Control message flow rate

  • üîÑ Reset State Node - Clear rate limit history

üîó How State Nodes Work Together

The Two-Part System

State nodes work in two parts:

  1. State Nodes - Hold the actual state (ON/OFF, count, etc.)
  2. Action Nodes - Control and modify the state
You must add a state node FIRST before you can control it with actions!

Step-by-Step Example

  1. Add a üö¶ ON/OFF Switch node to your flow
  2. Give it a name like "Cooldown Switch"
  3. Connect your trigger through this switch
  4. Add a üö¶ Set Gate State action after your main action
  5. Configure it to turn the "Cooldown Switch" OFF
  6. Add a delay
  7. Add another üö¶ Set Gate State to turn it back ON

üí° Pro Tips

Name your state nodes clearly! Use names like "Hello Cooldown", "Viewer Counter", "Points Total" instead of generic names.
State persists between messages. A switch stays OFF until you turn it ON. A counter keeps counting until you reset it.
Counters auto-increment. When messages pass through a Counter node, it automatically adds 1. Use "Increment Counter" action to add different amounts.
Test with low numbers first. Set counters to trigger at 2 or 3 while testing, then increase for production.
